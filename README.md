## Task 1: AI-Powered Code Completion

**Objective:**  
Use AI-powered code completion tools such as **GitHub Copilot** or **Tabnine** to implement a Python function that sorts a list of dictionaries by a specific key.

**Implementation Details:**  
- Manual implementation vs AI-assisted implementation  
- Comparison of efficiency, readability, and maintainability  

**Key Takeaways:**  
AI-assisted code completion accelerates development by providing syntactically correct suggestions, reducing the likelihood of errors, and suggesting optimized code patterns. The manual implementation allows deeper understanding of logic but is slower to write and verify.

**Deliverables:**  
- Python scripts (`task1_ai_code.py`)  
- 200-word analysis embedded in code comments

## Task 2: Automated Testing with AI

**Objective:**  
Automate a login page test case using Selenium and demonstrate AI-assisted testing benefits. The tests include both valid and invalid credentials.

**Implementation Details:**  
- Used **Selenium WebDriver** in headless Chrome mode  
- Test cases include:  
  1. Valid username and password (success expected)  
  2. Invalid username (failure expected)  
  3. Invalid password (failure expected)  
- Captured test results in a structured table using **Pandas**  
- Screenshot of test results saved for reporting  

**Benefits of AI in Testing:**  
- Generates additional test scenarios to cover edge cases  
- Reduces manual effort and human errors  
- Provides predictive insights into potential failure points  

**Deliverables:**  
- Python test script (`login_test.py`)  
- Screenshot of test results (`login_test_results.png`)  
- 150-word summary explaining AI advantages in test coverage
